# Деплой Telegram-бота на Amvera

## Важно про токен

Не вставляй токен Telegram-бота в код и не коммить его в GitHub. Токен должен храниться только в переменных окружения на хостинге.

## Что уже подготовлено

В проекте есть `Dockerfile`, поэтому Amvera может собрать и запустить проект как Docker-приложение.

Команда запуска внутри контейнера:

```bash
python telegram_bot/bot.py
```

## Переменные окружения

В панели Amvera добавь переменные:

```text
BOT_TOKEN=твой_токен_от_BotFather
MAP_URL=https://wwgame28.github.io/perimeter/web_map/
```

`BOT_TOKEN` — обязательная переменная. Без неё бот не запустится.

`MAP_URL` — ссылка на мини-карту. Можно оставить как есть, если GitHub Pages будет включён для репозитория.

## Порядок запуска

1. Создай проект на Amvera.
2. Подключи GitHub-репозиторий:

```text
wwgame28/perimeter
```

3. Выбери сборку через Dockerfile.
4. Добавь переменные окружения `BOT_TOKEN` и `MAP_URL`.
5. Запусти деплой.
6. Открой Telegram и отправь боту:

```text
/start
/play
/map
```

## Если бот не отвечает

Проверь:

- правильно ли указан `BOT_TOKEN`;
- нет ли пробелов в начале или конце токена;
- запустился ли контейнер без ошибок;
- видит ли Amvera файл `Dockerfile`;
- есть ли в логах ошибка `Set BOT_TOKEN environment variable`.

## Почему токен не в коде

Репозиторий публичный. Если вставить токен в код, любой человек сможет управлять ботом. Поэтому токен хранится только в панели Amvera как секретная переменная окружения.
