# HTML Map System

## Цель

HTML-карта нужна, чтобы игрок видел, где сейчас находятся герои и какие зоны комплекса уже открыты. Она не должна раскрывать секреты сюжета, но должна усиливать ощущение живого объекта.

## Файлы

```text
web_map/
├── index.html
├── style.css
└── app.js
```

## Визуальный стиль

Карта сделана в стиле мрачной доски/полевого плана:

- тёмный фон;
- белые линии, похожие на мел;
- красные опасные зоны;
- маркеры персонажей;
- панель состояния команды.

## Состояния карты

В `web_map/app.js` есть объект `STATES`. Каждый state описывает:

- день;
- время;
- позиции персонажей;
- статус каждого героя.

Пример ссылки:

```text
https://wwgame28.github.io/perimeter/web_map/?state=day2_missing
```

## Интеграция с Telegram

В Telegram-боте нужно добавить команду `/map`, которая отправляет игроку ссылку на карту.

Если включить GitHub Pages, карта будет доступна примерно по адресу:

```text
https://wwgame28.github.io/perimeter/web_map/
```

Для WebApp-кнопки Telegram нужен HTTPS URL. GitHub Pages подходит.

## Как обновлять карту по сюжету

Когда игрок переходит в новую сцену, бот может отправлять ссылку с параметром:

```text
/map → web_map/?state=day1_yard
```

Примеры готовых состояний:

- `day1_start` — команда у КПП;
- `day1_yard` — первое разделение во дворе;
- `day2_missing` — утро второго дня, Глеб пропал;
- `day3_final` — финальная лаборатория.

## Следующий шаг

Добавить в движок соответствие:

```python
SCENE_TO_MAP_STATE = {
    "start": "day1_start",
    "yard": "day1_yard",
    "day2_start": "day2_missing",
    "final_lab": "day3_final",
}
```

Тогда карта будет автоматически показывать актуальную ситуацию.
